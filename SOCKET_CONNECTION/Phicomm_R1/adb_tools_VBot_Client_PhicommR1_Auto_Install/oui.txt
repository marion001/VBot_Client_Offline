</div></div></div></div><h3 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h4 mt-5" >Registries</h3><div class="vc_row wpb_row vc_inner vc_row-fluid"><div class="wpb_column vc_column_container vc_col-sm-6"><div class="vc_column-inner"><div class="wpb_wrapper"><h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5" >MAC Address Block Large (MA-L)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>An MA-L assignment includes an OUI and large blocks of EUI-48 and EUI-64 values which can be used as MAC Addresses, Bluetooth Device Addresses, Ethernet Addresses or object identifiers.</p>
<p><a class="arrow-link" href="/products-programs/regauth/oui/">Learn More About MA-L</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >MAC Address Block Medium (MA-M)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>An MA-M assignment includes medium blocks of EUI-48 and EUI-64 values which can be used as MAC Addresses, Bluetooth Device Addresses, Ethernet Addresses or object identifiers.</p>
<p><a class="arrow-link" href="/products-programs/regauth/oui28/">Learn More About MA-M</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >MAC Address Block Small (MA-S)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>An MA-S assignment includes an OUI-36 and small blocks of EUI-48 and EUI-64 values which can be used as MAC Addresses, Bluetooth Device Addresses, Ethernet Addresses or object identifiers.</p>
<p><a class="arrow-link" href="/products-programs/regauth/oui36/">Learn More About MA-S</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Company ID (CID)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>A unique 24-bit identifier that cannot be used to generate EUI-48 or EUI-64 values. Therefore, the CID is especially applicable in applications where unique MAC addresses are not required.</p>
<p><a class="arrow-link" href="/products-programs/regauth/cid/">Learn More About CID</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >EtherType™</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>Provides a context for interpretation of the data field of an Ethernet/IEEE 802.3 data frame (protocol identification).</p>
<p><a class="arrow-link" href="/products-programs/regauth/ethertype/">Learn More About EtherType™</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Manufacturer ID</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>The manufacturer of a transducer is defined with a 14-bit code called the manufacturer ID.</p>
<p><a class="arrow-link" href="/products-programs/regauth/manid/">Learn More About Manufacturer ID</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Logical Link Control (LLC)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>Contains addressing information that consists of the Destination Service Access Point (DSAP) and the Source Service Access Point (SSAP) address field.</p>
<p><a class="arrow-link" href="/products-programs/regauth/llc/">Learn More About LLC</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >AcpcTreeId</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>An identifier for a specific set of binary trees used to convey certificate access codes to certificate users who are using the Activation Codes for Pseudonym Certificates (ACPC) technique specified in IEEE Std 1609.2.1. The assignee of an AcpcTreeId is assigned to operate an ACPC Certificate Access Manager using that AcpcTreeId.</p>
<p><a class="arrow-link" href="/products-programs/regauth/acpctid/">Learn More About AcpcTreeId</a></p>

		</div>
	</div>
</div></div></div><div class="wpb_column vc_column_container vc_col-sm-6"><div class="vc_column-inner"><div class="wpb_wrapper"><div class="vc_empty_space  hide-desktop"   style="height: 3rem"><span class="vc_empty_space_inner"></span></div><h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5" >Unique Registration Number (URN)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>A 64-bit unique identifier contained in memory devices, or nodes, in which IEEE 1451.4 TEDS data is stored.</p>
<p><a class="arrow-link" href="/products-programs/regauth/urn/">Learn More About URN</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >IEEE Template/TDL Item</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>IEEE template is a definition of the placement and significance of each piece of data stored within the TEDS memory. A TDL&#8217;s purpose is to map bits.</p>
<p><a class="arrow-link" href="/products-programs/regauth/tdl/">Learn More About IEEE Template/TDL Item</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Provider Service Identifier (PSID)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>Used by IEEE 1609 standards to identify a particular application service provider that announces it is providing a service to potential users of an application or service.</p>
<p><a class="arrow-link" href="/products-programs/regauth/psid/">Learn More About PSID</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >IEEE 802.16 Operator ID</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>A 24-bit number that is broadcast by a base station as part of its Base Station ID.</p>
<p><a class="arrow-link" href="/products-programs/regauth/bopid/">Learn More About IEEE 802.16 Operator ID</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Standard Group MAC Address</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>NOTE: For Standards Development use ONLY. A Universally Administered Address Block allocated for the assignment of Group MAC Addresses for use in Standards.</p>
<p><a class="arrow-link" href="/products-programs/regauth/grpmac/">Learn More About Standard Group MAC Address</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >IEEE 1667 Silo Type Identifier (STID)</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>IEEE 1667 specifies the interface to and behavior of several silos (e.g., Probe Silo, TCG Storage Transport Silo, Password Silo, and Smart Card Transport Silo). External silos may be defined outside of the standard, be either public or private, and involve third-party licensing. Guidelines for external silos are specified in IEEE 1667.</p>
<p><a class="arrow-link" href="/products-programs/regauth/stid/">Learn More About STID</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >IEEE Std 1588™ sdoId</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>The sdoId (Standards Development Organization Identifier) is a protocol attribute, per IEEE Std 1588, that identifies the PTP Profile in use by the PTP Instance.</p>
<p><a class="arrow-link" href="/products-programs/regauth/sdoid/">Learn More About IEEE Std 1588™ sdoId</a></p>

		</div>
	</div>
<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >CtlSeriesId</h4>
	<div class="wpb_text_column wpb_content_element" >
		<div class="wpb_wrapper">
			<p>An identifier for a Certificate Trust List series allowing for the management of roots of trust within the certificate management system as specified in IEEE Std 1609.2.1. The assignee of an CtlSeriesId serves as the SCMS Manager of that CtlSeriesId.</p>
<p><a class="arrow-link" href="/products-programs/regauth/ctlseriesid/">Learn More About CtlSeriesId</a></p>

		</div>
	</div>
</div></div></div></div><h3 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h4 mt-5" >Registration Authority Governance and Additional Information</h3>

		<div class="wpb_wrapper">
			<p>The IEEE Registration Authority operates under the direction of the <a href="/about/bog/">IEEE SA Board of Governors</a>.</p>

		</div>

<h4 style="text-align: left" class="vc_custom_heading vc_do_custom_heading h5 mt-5" >Registration Authority Committee (RAC)</h4>

<p><a class="arrow-link" href="/about/bog/rac/">Learn More About the Registration Authority Committee (RAC)</a></p>
